安装

安装方法

插件安装优先；rules 仍需单独安装到 .factory 目录。

前置条件
建议 Factory Droid CLI v2.1.0+。先确认版本：
droid --version
方式一：作为插件安装（推荐）
/plugin marketplace add affaan-m/everything-factory-droid
/plugin install everything-factory-droid@everything-factory-droid
插件安装后即可使用 commands / agents / skills，但 rules 不会随插件自动分发，需要单独执行下面的安装步骤。
方式二：克隆仓库后安装 rules
git clone https://github.com/affaan-m/everything-factory-droid.git
cd everything-factory-droid
npm install

# macOS / Linux
./install.sh --profile full
# 或按语言选择
./install.sh typescript python golang swift php

# Windows PowerShell
npx efd-install --profile full
# 或按语言选择
npx efd-install typescript python golang swift php
手动复制 rules（理解目录结构时很有用）
mkdir -p ~/.factory/rules
cp -r rules/common ~/.factory/rules/
cp -r rules/typescript ~/.factory/rules/
cp -r rules/python ~/.factory/rules/
cp -r rules/golang ~/.factory/rules/
cp -r rules/web ~/.factory/rules/
复制 rules 时要复制整个目录，例如 rules/common、rules/typescript。不要使用 /* 展平，否则会破坏相对引用并导致同名文件互相覆盖。
常用环境变量
# Hook profile: minimal | standard | strict
export EFD_HOOK_PROFILE=standard

# 临时禁用若干 hook ID
export EFD_DISABLED_HOOKS="pre:edit:create:config-protection,post:edit:console-warn"
开始

快速开始

3 分钟建立对 EFD 主工作流的直觉。

先记住一件事
EFD 当前的主工作流面是 skills；commands 主要是兼容入口。你可以继续用斜杠命令，但在编写新工作流时应该优先放到 skills/。
# 插件安装后的 namespaced 入口
/everything-factory-droid:plan "新增认证系统"

# 手动安装时可以直接用短命令
/plan "新增认证系统"
/tdd
/code-review
/verify
/save-session
会话数据默认保存到 ~/.factory/session-data/。恢复时可用 /resume-session 或 /sessions。
如果你要用 multi-* 系列命令，还需要先运行 npx ccg-workflow 安装对应运行时。

原则

核心原则

如果你只记住少数几个要点，请记住这 6 条。

技能优先
EFD 的主工作流面是 skills；commands 主要用于兼容与过渡。
先计划再执行
复杂工作先用 /plan 或 blueprint 明确范围、风险和步骤。
测试驱动
默认按 TDD 和 verification-loop 思路推进。
安全前置
涉及输入、身份、密钥、支付、部署时先做安全审查。
上下文节制
合理使用 compact、context-budget、token-budget-advisor 控制成本。
持续学习
通过 learn / evolve / skill-create 把经验沉淀为资产。
